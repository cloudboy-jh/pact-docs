This guide will get you from zero to a fully synced development environment.

## Overview

Pact works like `git` — it creates a `.pact/` folder in your current directory that contains all your configuration files. Your GitHub token is stored globally in your OS keychain.

Pact commands use git-like directory resolution: they check the current directory for `.pact/`, walk up the directory tree, and fall back to `~/.pact/`.

## 1. Install Pact

```bash
brew install cloudboy-jh/tap/pact
```

## 2. Initialize Pact

Run `pact init` from wherever you want your `.pact/` folder to live:

```bash
# For global configs (available everywhere)
cd ~
pact init

# Or for project-specific configs
cd ~/projects/my-app
pact init
```

This will:
1. Open your browser for GitHub authentication
2. Create a `my-pact` repo in your GitHub account (if it doesn't exist)
3. Clone it to `./.pact/` in your current directory
4. Create a default `pact.json` configuration

## 3. Add your configurations

Edit your pact configuration:

```bash
pact edit                 # Opens pact.json in your $EDITOR
pact edit shell           # Edit shell configs
pact edit web             # Open web editor in browser
```

## 4. Sync your configs

```bash
pact sync
```

This will show an interactive module picker:

```txt
Found 6 modules in pact.json:

  [1] shell        2 files
  [2] editor       12 files (nvim, vscode)
  [3] git          2 files
  [4] ai           3 files (prompts, agents)
  [5] tools        1 file
  [6] theme        4 files (colors, wallpaper)

Select modules: 1,2,6
```

## 5. Check status

```bash
pact
```

This shows an interactive status display with quick actions.

## On a New Machine

When you're setting up a new machine:

```bash
cd ~                      # Or wherever you want .pact/ to live
pact init                 # Clones your existing pact repo to ./.pact/
pact sync                 # Applies all configs
```

Changes sync through GitHub — edit on one machine, pull on another.

## Directory Resolution

Pact finds your config using git-like tree walking. If you initialize in a parent directory, subdirectories automatically find it:

```bash
cd ~/projects/my-app
pact init                 # Creates ~/projects/my-app/.pact/

cd ~/projects/my-app/src/components
pact sync                 # Works! Finds .pact/ in parent directory
```

Resolution order:
1. Current directory
2. Parent directories (walking up)
3. Falls back to `~/.pact/`

## Bootstrap from Existing Setup

Already have a configured development environment? Use `pact read` to import your existing setup:

```bash
cd my-project
pact read                 # Scans your local environment
# Select which tools and configs to import...
# ✓ Updated pact.json
# ✓ Copied config files to .pact/

pact push -m "Initial import from macbook"
```

This detects your installed CLI tools, shell config, editor settings, git config, and more.

## Common Commands

| Command | Description |
|---------|-------------|
| `pact` | Interactive status with quick actions |
| `pact init` | Initialize pact in current directory |
| `pact sync` | Interactive sync with module picker |
| `pact sync shell` | Sync specific module |
| `pact read` | Scan local environment and import to pact |
| `pact read --diff` | Show drift between local and pact.json |
| `pact edit` | Edit pact.json in $EDITOR |
| `pact edit web` | Open web editor |
| `pact push` | Commit and push changes |
| `pact status` | Show status (non-interactive) |

## Example Workflow

Here's a typical workflow for managing your development environment:

```bash
# Start a new project
cd new-project
pact init

# Add some shell customizations
pact edit shell/zshrc     # Edit your shell config
pact push -m "Add custom aliases"

# On another machine
cd new-project
pact init                 # Clones same pact repo
pact sync shell           # Apply shell configs
```


Use `pact edit web` to edit your configurations in a beautiful web interface with syntax highlighting and visual editing.
